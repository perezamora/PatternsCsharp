## Ejemplos patrones diseño con C#

Recursos de internet para aprender a implementar patrones de diseño.

Principios SOLID:
  * S - SRP (Single responsibility principle).
	> Una clase debe encargarse de una sola responsabilidad.
  * O - OCP (Open closed principle).
	> La extensión debe preferirse a la modificación.
  * L - LSP (Liskov substitution principle).
	> Un objeto clase padre debería ser capaz de referir objetos secundarios sin problemas 
	> durante el polimorfismo en tiempo de ejecución.
  * I - ISP (Interface Segregation principle).
	> No se debe forzar al cliente a usar una interfaz si no la necesita.
  * D - Dependency inversion principle.
	> Los módulos de alto nivel no deben depender de módulos de bajo nivel, sino que deben depender de la abstracción.

  ---

Ejemplos sencillos dondes se exponen algunos patrones de diseño.

  * Abstract Factory: 
	- Proporcione una interfaz para crear familias de objetos relacionados o dependientes sin especificar sus clases concretas.
